# HM Land Registry Operations and Demand Analytics

## Portfolio Summary

This project analyzes HM Land Registry application, transaction, search, customer, local authority, and regional data to identify operational trends and workload drivers.

The work combines data profiling, quality assessment, exploratory analysis, KPI definition, customer concentration analysis, and dashboard recommendations. It is designed as a portfolio project that demonstrates both analytical thinking and reproducible data engineering.

## Business Questions

- Which service categories create the highest workload?
- How concentrated is demand across customers and regions?
- Which data quality issues affect reliable reporting?
- What KPIs should be monitored for operational performance?
- What dashboard views would help leaders track demand, risk, and service mix?

## Key Outputs

- Cleaned silver tables in `data/silver/`
- Analytics-ready gold tables in `data/gold/`
- Data validation evidence in `data/pipeline_validation.json`

- Engineering report in `Engineering_report.md` and `Engineering_report.html`

## Headline Findings

- Latest application volume: 1,541,143
- Latest transaction volume: 66,426
- Latest search volume: 210,935
- Mean data quality score: 97.16
- Largest service category: OC1
- Highest-volume region: South East
- Highest-volume customer: Enact
